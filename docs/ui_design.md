# AutoSubMaker 画面構成案

## 1. 画面方針

MVP では、`NiceGUI` のネイティブウィンドウを使った単一ウィンドウ構成を採用する。

- 初回起動時のみセットアップ導線を表示する
- 通常利用時は 1 画面で動画投入から処理開始まで完結する
- 環境設定やエラー詳細はダイアログで補助する

## 2. ウィンドウ構成

### 2.1 メインウィンドウ

役割:

- 動画の追加
- キューの確認
- 出力設定の変更
- 処理開始
- 進捗確認

推奨レイアウト:

- 上部: ヘッダー
- 中央左: 入力とキュー
- 中央右: 設定パネル
- 下部: 実行ログ

### 2.2 初回セットアップダイアログ

役割:

- FFmpeg 検出結果の表示
- FFmpeg 未検出時の取得案内
- Whisper モデル初回ダウンロードの案内
- アプリ管理ディレクトリの説明

表示タイミング:

- 初回起動時
- 必須依存が不足しているとき

### 2.3 エラー詳細ダイアログ

役割:

- 失敗したジョブのエラー詳細表示
- FFmpeg 実行失敗内容の表示
- モデル取得失敗理由の表示

## 3. メインウィンドウ詳細

### 3.1 ヘッダー

表示項目:

- アプリ名 `AutoSubMaker`
- FFmpeg 状態バッジ
- Whisper モデル状態バッジ
- 出力先表示
- `環境設定` ボタン

状態バッジ例:

- `FFmpeg: Ready`
- `FFmpeg: Missing`
- `Whisper Model: Ready`
- `Whisper Model: Downloading`

### 3.2 入力カード

目的:

- 動画追加を最短で行う

配置要素:

- `動画を追加` ボタン
- `複数動画を追加` ボタン
- ドラッグアンドドロップ領域
- 追加済み件数表示

MVP では見送る項目:

- フォルダ単位読み込み
- 直近履歴

### 3.3 キューテーブル

目的:

- 処理対象の一覧管理

カラム案:

- 状態
- ファイル名
- 解像度
- 向き
- 長さ
- 出力モード
- 進捗
- 出力先

行ごとの操作:

- 削除
- 再実行
- エラー詳細表示

状態表示:

- 待機中
- 実行中
- 完了
- 失敗

### 3.4 実行コントロール

配置要素:

- `処理開始` ボタン
- `完了済みをクリア` ボタン
- `失敗分を再試行` ボタン

MVP では未搭載でもよい項目:

- 一時停止
- 全体キャンセル

### 3.5 設定パネル

設定パネルはメイン画面右側の固定カラムとする。

#### 出力設定カード

- 出力先フォルダ
- 字幕ファイル出力の有無
- 焼きこみ動画出力の有無
- 出力ファイル名ルール
- 一時ファイル保持の有無

#### 文字起こし設定カード

- 言語モード
  - 自動判定
  - 固定指定
- モデルサイズ
- 実行デバイス
  - Auto
  - GPU 優先
  - CPU 固定

#### 字幕整形設定カード

- 横長動画の最大文字数
  - 初期値 18
- 縦長動画の最大文字数
  - 初期値 13
- 最大行数
  - 初期値 2
- 最小表示時間
  - 初期値 0.8 秒
- 最大表示時間
  - 初期値 4.0 秒

#### 字幕スタイル設定カード

- フォント名
- フォントサイズ
- 文字色
- 縁取り色
- 縁取り太さ
- 下余白
- 横位置揃え

### 3.6 ログ表示エリア

目的:

- 現在の実行内容を見える化する

表示内容:

- 起動時依存チェック結果
- FFmpeg 実行ログ要約
- Whisper 実行ステータス
- 字幕生成完了通知
- 焼きこみ完了通知
- エラー要約

## 4. 初回起動フロー

1. 設定ファイルを読み込む
2. アプリ管理ディレクトリを作成する
3. FFmpeg の存在確認を行う
4. FFmpeg 未検出時は取得または手動指定を案内する
5. Whisper モデルの存在確認を行う
6. モデル未取得時はダウンロードを開始する
7. 準備完了後にメイン画面へ遷移する

## 5. ユーザーフロー

### 5.1 通常フロー

1. 動画を追加する
2. 出力先と設定を確認する
3. `処理開始` を押す
4. キュー進捗を確認する
5. 出力フォルダで結果を確認する

### 5.2 FFmpeg 未検出フロー

1. 起動時に FFmpeg 未検出を表示する
2. ユーザーが `自動ダウンロード` または `手動指定` を選ぶ
3. 準備完了後に通常フローへ戻る

### 5.3 モデル未取得フロー

1. 起動時にモデル未取得を表示する
2. 自動ダウンロードを開始する
3. ダウンロード中は進捗を表示する
4. 完了後に通常フローへ移行する

## 6. レスポンシブ方針

ネイティブウィンドウ前提だが、最小幅では上下配置へ切り替えられる構成が望ましい。

- 通常幅: 左右 2 カラム
- 狭い幅: 上にキュー、下に設定

## 7. MVP で入れない画面

- 字幕プレビュー専用画面
- タイムライン編集画面
- 話者編集画面
- 翻訳設定画面

## 8. デフォルトウィンドウ案

- 初期サイズ: `1440 x 960`
- 最小サイズ: `1100 x 760`
- リサイズ可

この構成であれば、MVP の主要操作を 1 画面に集約しつつ、初回セットアップと運用時の見通しを両立しやすい。
