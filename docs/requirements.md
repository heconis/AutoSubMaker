# AutoSubMaker 要件整理

## 1. 目的

AutoSubMaker は、ローカル環境で動画から音声を抽出し、音声認識によって字幕を生成し、字幕を動画へ自動で焼きこむデスクトップ向けツールとする。

想定する基本フローは以下のとおり。

1. 動画を読み込む（単体または複数）
2. FFmpeg で音声を抽出する
3. ローカル Whisper で文字起こしする
4. 字幕ファイルを生成する
5. FFmpeg + ASS で字幕を動画に焼きこむ

## 2. 想定ユーザー

- ローカル環境で完結した字幕付けをしたい個人ユーザー
- YouTube や SNS 向けに字幕入り動画を量産したいユーザー
- API 利用料をかけずに自動字幕を作りたいユーザー

## 3. MVP の範囲

MVP では、まず「動画を投入すれば字幕焼き込み済み動画を出力できる」ことを最優先とする。

### MVP に含める

- 単体動画の処理
- 複数動画の連続処理
- ローカル Whisper による文字起こし
- SRT の生成
- ASS の生成
- 字幕ファイル単体出力
- ASS を使った字幕焼き込み動画の出力
- 出力先フォルダの指定
- 処理進捗とエラー表示

### MVP では見送る

- 話者分離
- 自動翻訳
- 字幕の手動編集 UI
- タイムライン編集
- BGM とセリフの自動分離
- クラウド連携

## 4. 推奨する前提条件

要件を固めるため、現時点では以下を前提にすることを推奨する。

- 対応 OS は Windows を最優先とする
- 実装言語は Python
- GUI アプリとして提供する
- GUI フレームワークは `NiceGUI` を採用し、ネイティブウィンドウで提供する
- 音声認識エンジンは `faster-whisper` を第一候補とする
- FFmpeg は初期状態では同梱せず、既存インストールの検出またはアプリ経由の取得で利用する
- 出力形式は「字幕ファイル」と「字幕焼き込み済み動画」の両方を残す

## 5. 主要ユースケース

### UC-01: 単体動画から字幕付き動画を作る

1. ユーザーが動画ファイルを選択する
2. システムが音声抽出を実行する
3. システムが文字起こしを実行する
4. システムが字幕ファイルを生成する
5. システムが字幕を焼きこんだ動画を出力する

### UC-02: 複数動画を一括処理する

1. ユーザーが複数の動画ファイルを選択する
2. システムがキュー順に処理する
3. ファイル単位で成功・失敗を記録する
4. 失敗したファイルがあっても、残りの処理は継続する

### UC-03: 字幕ファイルだけを保存する

1. ユーザーが動画を選択する
2. システムが音声抽出と文字起こしを実行する
3. システムが SRT / ASS を出力する
4. 焼きこみはスキップできる

## 6. 機能要件

### 6.1 入力

- 対応入力形式は最低でも `mp4`, `mov`, `mkv` を含む
- 単一ファイル選択と複数ファイル選択に対応する
- ドラッグアンドドロップ、またはファイル選択ダイアログを提供する
- ファイルごとに以下を表示する
  - ファイル名
  - 長さ
  - 処理状態
  - 出力先

### 6.2 音声抽出

- FFmpeg を用いて入力動画から音声を抽出する
- Whisper に適した中間音声を生成する
- 推奨中間形式は `wav`, `16kHz`, `mono`
- 一時ファイルは処理完了後に削除できる
- デバッグ用途として保持設定も用意できる

### 6.3 依存ツール検出と初期セットアップ

- 起動時に FFmpeg と Whisper 実行要件の利用可否を確認する
- まず既定の探索先を確認する
  - 設定済み保存パス
  - アプリ管理ディレクトリ
  - `PATH` 環境変数上の実行ファイル
- FFmpeg が見つからない場合は以下を案内する
  - 実行ファイルのパス指定
  - 自動ダウンロード
- FFmpeg の配布は同梱ではなく、必要時取得を標準方針とする
- FFmpeg の自動取得対象は Windows 向け `essentials` ビルドを基本とする
- FFmpeg の自動取得先はアプリ管理ディレクトリとする
- Whisper については「実行ライブラリ本体」と「モデル」を分けて扱う
- ライブラリ未導入時はエラー内容を明示する
- モデルは初回起動時に自動ダウンロードする
- モデルの保存先はアプリ管理ディレクトリ配下を基本とする
- 設定したパスやモデル保存先は次回起動時に再利用する
- オフライン利用を考慮し、自動取得に失敗しても手動指定に切り替えられる

### 6.4 文字起こし

- ローカル Whisper で文字起こしを行う
- 対応モードは以下を想定する
  - 言語自動判定
  - 言語固定
- モデルサイズを選択できる
  - 例: `tiny`, `base`, `small`, `medium`, `large`
- GPU が利用可能な場合は GPU を優先し、利用不可なら CPU にフォールバックする
- ファイル単位で文字起こし結果を保持する

### 6.5 字幕生成

- 文字起こし結果から SRT を生成する
- 焼きこみ用に ASS を生成する
- 字幕ファイルのみを出力するモードを持つ
- 字幕整形ルールを持つ
  - 1 行あたりの最大文字数
    - 横長動画 `1920x1080` 系は 18 文字
    - 縦長動画 `1080x1920` 系は 13 文字
  - 1 字幕あたりの最大行数は 2 行
  - 表示時間の最小値 / 最大値
    - 最小 0.8 秒
    - 最大 4.0 秒
  - 句点 `。`、読点 `、`、空白、短い無音区間を優先して分割する
  - 助詞だけが行頭に来にくいように調整する
  - 句読点や閉じカッコが行頭に来にくいように調整する
  - 不自然に長い字幕は文節寄りで自動分割する
- 句読点や空白の整形を行う

### 6.6 字幕スタイル

- ASS のスタイル設定を持つ
- MVP では最低でも以下を設定可能にする
  - フォント名
  - フォントサイズ
  - 文字色
  - 縁取り色
  - 縁取り太さ
  - 画面下からの余白
  - 横位置揃え
- スタイルはプリセットとして保存できると望ましい

### 6.7 字幕焼きこみ

- FFmpeg を用いて ASS 字幕を動画に焼きこむ
- 元動画は上書きせず、別名で出力する
- 焼きこみ処理は任意でスキップできる
- 出力ファイル名の規則を設定できる
  - 例: `sample_subtitled.mp4`
- エンコード設定は最低限以下を選べる
  - 品質優先
  - 速度優先
- 可能であればハードウェアエンコードを利用できる

### 6.8 バッチ処理

- 複数動画をキューとして管理できる
- 各ジョブの状態を持つ
  - 待機中
  - 実行中
  - 完了
  - 失敗
  - キャンセル
- 一括開始、一時停止、キャンセルは将来的に対応候補とする
- MVP では「順番に実行」と「失敗しても次へ進む」を優先する

### 6.9 ログ・進捗表示

- 全体進捗とファイル単位進捗を表示する
- 現在の処理段階を表示する
  - 音声抽出中
  - 文字起こし中
  - 字幕生成中
  - 焼きこみ中
- エラー時は原因の要約を表示する
- 詳細ログをファイル保存できる

## 7. 非機能要件

### 7.1 オフライン性

- インターネット接続なしでも主要機能が動作する
- 音声データや字幕データを外部送信しない

### 7.2 操作性

- 動画選択から処理開始までを数クリックで完了できる
- 初回利用でも迷わないシンプルな UI とする
- よく使う設定は次回起動時に復元する

### 7.3 保守性

- 音声抽出、文字起こし、字幕整形、焼きこみをモジュール分割する
- 将来、翻訳や話者分離を追加しやすい構成にする

### 7.4 信頼性

- 1 件の失敗で全バッチが停止しない
- 中間ファイル削除失敗などの軽微なエラーは致命傷にしない
- 異常終了時に再実行しやすいログを残す

## 8. 入出力仕様

### 入力

- 動画ファイル
- 任意で設定ファイルまたは前回設定

### 中間生成物

- 抽出音声ファイル
- Whisper のセグメント結果
- `srt`
- `ass`

### 出力

- 字幕ファイル
  - `.srt`
  - `.ass`
- 字幕焼きこみ済み動画
- 処理ログ

## 9. 外部依存

- Python 実行環境
- NiceGUI
- FFmpeg
- Whisper 実行ライブラリ
  - 第一候補: `faster-whisper`
- 必要に応じて GPU 実行ライブラリ
- モデル配布元へのアクセス環境、またはローカルモデル配置

## 10. 想定アーキテクチャ

モジュールは以下の分割を想定する。

1. UI 層
2. ジョブ管理層
3. FFmpeg ラッパー層
4. Whisper 実行層
5. 字幕整形層
6. 字幕焼きこみ層
7. 設定 / ログ管理層

## 11. 受け入れ条件

以下を満たせば MVP 完了とみなす。

1. ユーザーが 1 本以上の動画を選択できる
2. 音声抽出から文字起こし、字幕生成、焼きこみまで自動実行できる
3. 出力先に `.srt`, `.ass`, 字幕付き動画が生成される
4. 失敗したファイルはエラー表示され、他ファイルは継続処理される
5. 最低限の字幕スタイル変更ができる
6. 焼きこみをスキップして `.srt` / `.ass` のみを出力できる

## 12. 現時点の推奨判断

まずは以下の構成で始めるのが最も現実的。

- Windows 向け Python デスクトップアプリ
- GUI は `NiceGUI` のネイティブウィンドウ構成
- 音声認識は `faster-whisper`
- 字幕出力は `SRT + ASS`
- 焼きこみは FFmpeg
- 入力は単体 / 複数ファイル対応
- FFmpeg は同梱せず、「自動検出 + 手動指定 + 必要なら `essentials` を取得」を基本方針とする
- FFmpeg の自動取得先はアプリ管理ディレクトリに固定する
- Whisper モデルは初回起動時に自動取得し、アプリ管理ディレクトリに保存する
- MVP では字幕編集機能は持たない

この方針であれば、最短で「使えるもの」を作りつつ、将来的な拡張にもつなげやすい。
